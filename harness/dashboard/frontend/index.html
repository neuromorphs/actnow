Object tracker

Runs dvs_track on the core and overlays its box on the raw event tap. The centre is the median event location and the box is sized by the spread (median absolute deviation) of the last Window N events around it. Gate radius bounds how far from the centre an event may count; Correlation is the spatio-temporal noise filter (0 = off, 2 = jAER default). All take effect when you load the tracker.

Reject a window as noise unless it collected at least this many events — filters flickery low-activity boxes. Applies live; no reload needed.

Raw tap display

The raw tap is accumulated into a per-pixel histogram over the persistence window. Denoise is how many of a pixel's 8 neighbours must be active for it to show (hides isolated and hot pixels) — lower it in low light, where real events are sparse; 0 = off. Density is the event count at which a pixel reaches full opacity and saturation.
